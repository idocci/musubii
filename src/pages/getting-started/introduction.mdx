# Introduction

MUSUBii（むすびー）は [Yaku Han JP](https://yakuhanjp.qranoko.jp/) を開発しているデザイナーの [クラク](https://twitter.com/Qrac_JP) が作った CSS フレームワークです。マークアップを覚え始めた 2016 年初頭から自分用に開発を始めました。

デザインや機能面のコーディングがフレームワークに依存しないよう、大きなコンポーネント・JavaScript は含まず「リセット CSS + ベース CSS」というミニマムな構成を取っています。

MUSUBii を使うだけでサイトは完成しませんが、毎度同じような CSS を書いている時間・クロスブラウザでチェックする時間を大幅に減らすことができ、疲弊することなくデザイン・コーディングを進められます。

日本の Web デザイナーが腹ごなしに MUSUBii を使い、元気にデザインしてくれたら幸いです。

## How It Works

MUSUBii は SCSS を用いたインブラウザデザインを想定しています。インブラウザデザインとは、デザインとコーディングを同時に行いブラウザ上で仕上がりを確認しながらデザインしていく手法です。

開発者である [クラク](https://twitter.com/Qrac_JP) は、Web やアプリの UI デザインにおいて XD・Illustrator・Photoshop の代わりにコーディングをデザインツールとして使っています。

Web デザインはコーディングしないと完成しません。であれば、最初からコーディングがデザインを兼ねていれば一石二鳥です。ただし、そのためにはコーディングが早く柔軟でないとなりません。

そこで、よく使う且つカスタマイズしやすい SCSS をまとめた MUSUBii を使います。

## After...

余談ですが、開発初期にリセット CSS の [Shitaji.CSS](https://shitajicss.qranoko.jp/)、色を考えているときに [MOFTONE](https://mixtone.qranoko.jp/) などのカラーセット、 SCSS 変数をまとめて外部化した [Sass Dashi](https://github.com/qrac/sass-dashi) が生まれました。

v6 以前は各ライブラリに役割を分担していましたが、コードの把握が難しくなったため、v7 以降は MUSUBii 内で全てのコードを管理しています。
